# 骑手申请 & 聊天系统 — 后端接口文档

> 前端工程：PandadeliveroWebsite（官网）
> API 基础路径：`https://demo.pandadelivero.com/api/v1/`（Chat 服务）和 `https://demo.pandadelivero.com/`（Staff 服务）
> 请求格式：FormData（`application/x-www-form-urlencoded`）
> 响应格式：`{ error: "0" }` 为成功，数据在 `data` 字段；否则 `{ error: "1", message: "错误信息" }`
> Chat 服务接口均走 `/api/v1/`（不走独立服务域名）

---

## 零、前置说明

### 响应字段路径约定

后端所有接口的响应统一包装为 `{ error, data }` 格式。
前端 axios 默认配置会将响应透传给 `res`，即：

```javascript
// 实际 res 结构
{
  error: "0",
  data: { ... }   // 实际数据在这里
}

// 取数据应始终使用 res.data.XXX
// 不应直接使用 res.XXX（除非后端明确返回顶层字段）
```

> **前端注意事项**：如果 axios 配置了响应拦截器自动解包 `res.data`（即拦截器返回 `response.data`），则代码中可直接用 `res.XXX`。请与后端确认当前拦截器行为。所有页面已按兼容两种模式实现（先检查 `res.data`，再降级到 `res`）。

#### 失败响应

```json

  "message": "Token 已失效，请重新从后台获取链接"
```

#### Token 生成规则（后端实现参考）

```python
import hmac
import time
import base64
import json

SECRET_KEY = "your-server-secret-key"
EXPIRY_HOURS = 24

def generate_recruiter_token(recruiter_id: int, recruiter_name: str) -> str:
    """
    生成运营人员访问 token
    """
    expiry = int(time.time()) + EXPIRY_HOURS * 3600
    raw = f"{recruiter_id}|{recruiter_name}|{expiry}"
    sig = hmac.new(
        SECRET_KEY.encode(),
        raw.encode(),
        'sha256'
    ).hexdigest()
    payload = f"{raw}|{sig}"
    return base64.b64encode(payload.encode()).decode()


def verify_recruiter_token(token: str) -> dict | None:
    """
    验证并解析 token
    返回 None 表示无效
    """
    try:
        raw = base64.b64decode(token.encode()).decode()
        recruiter_id, recruiter_name, expiry, sig = raw.rsplit('|', 3)

        # 检查过期
        if int(expiry) < int(time.time()):
            return None

        # 验证签名
        expected_sig = hmac.new(
            SECRET_KEY.encode(),
            f"{recruiter_id}|{recruiter_name}|{expiry}".encode(),
            'sha256'
        ).hexdigest()
        if sig != expected_sig:
            return None

        return {
            "recruiter_id": int(recruiter_id),
            "recruiter_name": recruiter_name
        }
    except Exception:
        return None
```

> **前端注意事项**：token 内容由后端生成和验证，前端只做 URL 透传，不解析、不存储、不修改。

**说明**：运营人员输入中文，调用此接口生成西语预览。**翻译由 GPT 实现**（后端调用 OpenAI GPT API，前端只调用接口）。

#### 请求体

|---|---|---|---|
| text | string | 是 | 待翻译文本（中文） |
| source_lang | string | 是 | 源语言：`zh` |
| target_lang | string | 是 | 目标语言：`es` |

#### 成功响应

```json
{
  "error": "0",
  "data": {
    "translated_text": "¿Estás dado de alta como autónomo?",
    "source_lang": "zh",
    "target_lang": "es"
  }
}
```

#### GPT 集成说明（后端实现参考）

```
模型建议：gpt-4o-mini（成本低、速度快）
API 端点：https://api.openai.com/v1/chat/completions

Prompt 示例（Python 伪代码）：

def translate_zh_to_es(text: str) -> str:
    response = openai.ChatCompletion.create(
        model="gpt-4o-mini",
        messages=[
            {
                "role": "system",
                "content": "You are a professional translator. Translate the following Chinese text to Spanish. Only return the Spanish translation, nothing else."
            },
            {
                "role": "user",
                "content": text
            }
        ],
        temperature=0.3,
        max_tokens=500
    )
    return response.choices[0].message.content.strip()
```

> **优化建议**：
> - 可在 Redis 中缓存近期翻译结果（key: MD5(text+source+target)），减少 API 调用
> - 请求频率限制：前端每次发送前调用一次，无并发需求，压力很小
> - 建议 timeout 设置为 10 秒，失败时前端显示"翻译失败，请重试"

## 三、运营人员后台操作流程

1. 运营人员登录 Pandadelivero 管理后台
2. 进入"骑手申请列表"页面
3. 找到需要联系的申请，点击「聊天」按钮
4. 后台调用 `generate_recruiter_token()` 生成 token
5. 拼接 URL：`https://Pandadelivero.com/admin-chat?token=<加密串>&application_id=PD-2026-00128`
6. 使用 `window.open(url, '_blank')` 打开新标签页
7. 官网 admin-chat 页面验证 token → 显示申请资料和聊天界面
8. 运营人员输入中文 → 点击"生成西语"预览 → 确认发送
